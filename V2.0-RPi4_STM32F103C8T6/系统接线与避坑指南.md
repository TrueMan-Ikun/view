# V2.0 硬件接线与避坑完全指南

在 V2.0 (Raspberry Pi 4 + STM32F103C8T6) 架构下，硬件连接看似简单，但因为涉及“多供电系统”以及“金属齿舵机”，极易因为接错线烧毁主板或引起系统复位崩溃。请严格按照以下图示与表格进行接线。

## 2. 接线引脚对照表

以常用的 STM32CubeMX 默认配置分配为例（你建立新工程时也可按需更改，以下为推荐接法）：

| 硬件模块 | 模块引脚 | 连接到目标 | 接线说明 |
| :--- | :--- | :--- | :--- |
| **USB 转 TTL** | TX | STM32 `PA3` (USART2_RX) | 树莓派发数据给单片机 |
| | RX | STM32 `PA2` (USART2_TX) | |
| | GND | STM32 `GND` | **必须要接，否则乱码** |
| **X轴舵机 MG90s**| 橙色/黄色 (信号) | STM32 `PA0` (TIM2_CH1) | 云台左右旋转 |
| | 红色 (VCC) | **独立外部开关电源 5V** | 见下方警告 |
| | 棕色/黑色 (GND) | STM32 `GND` 及 电源 `GND` | 共地 |
| **Y轴舵机 MG90s**| 橙色/黄色 (信号) | STM32 `PA1` (TIM2_CH2) | 云台上下俯仰 |
| | 红色 (VCC) | **独立外部开关电源 5V** | 见下方警告 |
| | 棕色/黑色 (GND) | STM32 `GND` 及 电源 `GND` | 共地 |
| **DFPlayer** | RX | STM32 `PB10` (USART3_TX) | 发送语音指令 (串接个1K电阻防杂音更好) |
| | TX | STM32 `PB11` (USART3_RX) | |
| | VCC | STM32 `5V` 引脚 | 工作电压要求严格，3.3V可能会声音沙哑 |
| **OLED (I2C)** | SCL | STM32 `PB6` (I2C1_SCL) | 时钟线 |
| | SDA | STM32 `PB7` (I2C1_SDA) | 数据线 |
| | VCC | STM32 `3.3V` 或 `5V`均可 | |

## 3. 核心注意事项（避坑指南）

> [!CAUTION]
> **永远不要用 STM32 开发板上的 5V 针脚直接带这俩只 MG90s 舵机！**
> 这是新手最最最常犯的错误！两只 MG90s 金属舵机在快速跟随时，启动瞬间会有 **1A ~ 2A 的堵转瞬态大电流**。如果你把舵机的红线插在 STM32 的 5V 上：
> 1. 大概率会瞬间把 STM32 主板供电芯片烧毁灭迹。
> 2. 会导致电压瞬间被拉成低于 3.3V，STM32 芯片直接黑屏重启，表现为“一动云台机子就死锁复位”。
> **正确做法：使用带锂电池的外部电压稳压板（降压到5V的BEC模块）单独给舵机供电，然后把电源的 GND 和 STM32 的 GND 连起来（共地）。**

> [!WARNING]
> **关于电压逻辑电平不匹配**
> 树莓派 4 的主板引脚是 **严格的 3.3V 耐压**。这就是为什么我强烈建议你使用廉价的 **USB 转 TTL 串口刷机线** 插在树莓派上来通讯，而不是直接用树莓派 GPIO 上的 TX/RX。使用 USB 给我们上了一层物理隔离保险，防止因为连线短路把昂贵的树莓派 4 给烧了。

> [!NOTE]
> **为什么必须要有那根“公用地线（GND 连通）”？**
> 单纯连 TX 和 RX 是无法通信的。只有两个设备的地线（基准 0V 电位）接通了，它们对 TX 和 RX 上 3.3V 的理解才是相同的标准。否则收发的数据全是各种诡异的乱码符号。只要两边连了数据线，就请务必拿一根杜邦线把两边的 GND 插在一块！

请仔细核对自己手头的杜邦线是否够用、降压板/电池是否安全。有任何关于电路的问题随时向我确认！
