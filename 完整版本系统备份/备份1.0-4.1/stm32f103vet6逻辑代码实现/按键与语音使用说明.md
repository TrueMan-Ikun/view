# 按键与语音使用说明

## 1. 当前按键对应关系

根据当前工程代码，5 个按键的功能如下：

| 按键功能 | GPIO 口线 | 当前作用 |
| --- | --- | --- |
| 厕所模式键 | `PC0` | 切换到厕所模式，先进入搜索，识别成功后播放 `0001.mp3` |
| 出口模式键 | `PC1` | 切换到出口模式，先进入搜索，识别成功后播放 `0002.mp3` |
| 办公室模式键 | `PE13` | 切换到办公室模式，先进入搜索，识别成功后播放 `0003.mp3` |
| 语音测试键 | `PE14` | 测试播放 / 暂停 / 继续 |
| 停止识别键 | `PE15` | 切换到停止识别模式，并播放 `0006.mp3` |

说明：

- `PE14` 不参与目标模式切换，只用于单独测试 DFPlayer。
- `PE14` 当前默认测试文件仍然是 `0001.mp3`，也就是“厕所在这”。
- 上电后默认识别模式是厕所模式，但系统不会自动播放模式语音。

## 2. 当前 MP3 文件编号对应内容

TF 卡内请按下面这套编号准备音频文件：

| 文件名 | 语音内容 | 当前状态 |
| --- | --- | --- |
| `0001.mp3` | 厕所在这 | 已接入 |
| `0002.mp3` | 安全出口在这 | 已接入 |
| `0003.mp3` | 办公室在这 | 已接入 |
| `0004.mp3` | 正在搜索目标 | 已接入 |
| `0005.mp3` | 已发现目标 | 已接入 |
| `0006.mp3` | 已停止识别 | 已接入 |
| `0007.mp3` | 目标丢失 | 已接入 |

## 3. TF 卡目录要求

当前代码使用的是 DFPlayer 的 `MP3` 文件夹播放命令，因此文件不能直接放在 TF 卡根目录，必须按下面结构放置：

```text
TF卡根目录
└─ MP3
   ├─ 0001.mp3
   ├─ 0002.mp3
   ├─ 0003.mp3
   ├─ 0004.mp3
   ├─ 0005.mp3
   ├─ 0006.mp3
   └─ 0007.mp3
```

建议：

- TF 卡格式使用 `FAT32`
- 文件名使用四位编号，不要用中文文件名
- 先确认每个 MP3 文件在电脑上都能正常播放

## 4. DFPlayer 接线说明

当前工程默认接线如下：

| 模块端 | 连接到哪里 | 说明 |
| --- | --- | --- |
| `VCC` | 稳定 `5V` | 建议使用稳定电源 |
| `GND` | STM32 `GND` | 必须共地 |
| `RX` | `PA9 (USART1_TX)` | 建议中间串 `1k` 电阻 |
| `TX` | 可不接 | 当前版本不读取模块返回状态 |
| `SPK1` | 小喇叭一端 | 直接外放 |
| `SPK2` | 小喇叭另一端 | 直接外放 |

按键接法：

- `PC0`、`PC1`、`PE13`、`PE14`、`PE15` 都是按键输入脚
- 每个按键建议一端接对应 GPIO，另一端接 `GND`
- 当前代码使用上拉输入，按下为低电平有效

## 5. 自动播报规则

当前系统已经接入以下自动语音：

### 模式切换播报

- 按 `PC0`：只切换到厕所模式，不立即播放 `0001.mp3`
- 按 `PC1`：只切换到出口模式，不立即播放 `0002.mp3`
- 按 `PE13`：只切换到办公室模式，不立即播放 `0003.mp3`
- 按 `PE15`：立即播放 `0006.mp3`
- 按键切换到厕所 / 出口 / 办公室后，会先进入一段“只允许寻找”的保护期
- 在这段保护期里，不会播“已发现目标”，也不会误播“目标丢失”

### 进入搜索播报

- 当系统进入搜索状态时，会自动播放 `0004.mp3`
- 切换到厕所 / 出口 / 办公室模式后，如果当前还没有找到目标，也会进入搜索流程
- 普通搜索场景下，`0004.mp3` 会在进入搜索后自动触发；如果系统仍处于语音保护期，则会稍后播放
- 目标真正丢失后的重捕获搜索分为两个阶段：
  - 阶段 1：局部快速搜索，会在 `0007.mp3` 之后允许补播一次 `0004.mp3`
  - 阶段 2：扩展为全范围搜索，不会重复播放 `0004.mp3`

### 首次发现目标播报

- 每次切换到厕所 / 出口 / 办公室模式后，第一次成功识别到目标时，会自动播放 `0005.mp3`
- 这里不是“看到一帧目标就播”，而是要连续稳定识别一小段时间后才会播报
- `0005.mp3` 播放后，系统会额外等待一段更充足的时间，再播放当前目标对应的名称语音，也就是 `0001.mp3 / 0002.mp3 / 0003.mp3`
- 这段等待时间里，云台仍会继续平滑校准，避免前一句还没说完就被后一条语音顶掉
- 同一模式下，后续即使丢失后再次找到，也不会重复播放 `0005.mp3`
- 切换到新的模式后，会重新允许播报一次 `0005.mp3`

### 目标丢失后重新找回

- 当系统原本已经锁定目标，后来确认目标丢失时，会播放 `0007.mp3`
- 丢失后的重新搜索不再是一上来全范围摇头，而是两阶段：
  - 先以最后一次有效位置为中心做小范围快速搜索
  - 若一段时间后仍未找回，再扩展为全范围搜索
- 丢失恢复流程里，阶段 1 允许补播一次 `0004.mp3`
- 进入阶段 2 后不再重复播放 `0004.mp3`
- 当目标重新稳定识别到时，不再播放 `0005.mp3`
- 重新找回后只播放当前目标对应的名称语音，也就是 `0001.mp3 / 0002.mp3 / 0003.mp3`

### None 模式规则

- 切换到 `None` 模式时播放 `0006.mp3`
- `None` 模式下不会再播放 `0004.mp3` 和 `0005.mp3`
- 原来尚未播出的搜索/发现语音也会被取消

## 6. 防乱播规则

为了避免语音连续打架，当前代码用了下面这套规则：

- 厕所 / 出口 / 办公室三个目标键不再立即播名称语音
- 搜索语音在进入搜索时触发；如果系统仍处于语音保护期，则会延后播放
- 如果搜索语音还没来得及播，目标就已经重新找到，则取消这条搜索语音
- “已发现目标”会先经过稳定确认，再决定是否播报
- 当前目标名称语音会排在“已发现目标”之后播放，并额外留出更充足的间隔，避免截断上一句
- 目标丢失后会先播 `0007.mp3`，随后只在局部快速重捕获阶段补播一次 `0004.mp3`
- 扩展到全范围搜索后，不会重复播放 `0004.mp3`
- 目标丢失后的重新找回流程中，仍然屏蔽 `0005.mp3`
- `0007.mp3` 在一次丢失恢复流程里只会播一次，不会因为画面轻微抖动连续重复
- “已发现目标”优先级高于“正在搜索目标”
- 同一模式下，“已发现目标”只播一次

## 7. 当前可以怎么测试

### 模式切换测试

1. 上电后等待约 1 秒，让 DFPlayer 完成初始化
2. 按 `PC0`，应只切换模式，不立即播放 `0001.mp3`
3. 按 `PC1`，应只切换模式，不立即播放 `0002.mp3`
4. 按 `PE13`，应只切换模式，不立即播放 `0003.mp3`
5. 按 `PE15`，应播放 `0006.mp3`
6. 切换到厕所 / 出口 / 办公室后的短时间内，不应立即听到“已发现目标”或“目标丢失”

### 搜索播报测试

1. 切换到厕所 / 出口 / 办公室模式
2. 保证当前没有识别到目标
3. 此时不应立即听到 `0001 / 0002 / 0003`
4. 系统进入普通搜索后，应听到 `0004.mp3`

### 发现目标播报测试

1. 切换到厕所 / 出口 / 办公室模式
2. 让系统先处于搜索状态
3. 再让 OpenMV 成功识别到目标
4. 连续稳定识别一小段时间后，应先播放一次 `0005.mp3`
5. `0005.mp3` 完整播完并留出一点余量后，再播放当前目标对应的名称语音
6. 同一模式下再次丢失又重新找到，不应重复播放 `0005.mp3`

### 目标丢失测试

1. 先让系统正常识别并锁定目标
2. 再让目标离开画面，直到系统确认丢失
3. 应先播放 `0007.mp3`
4. 丢失后的局部重捕获阶段，允许补播一次 `0004.mp3`
5. 如果阶段 1 没找回并进入阶段 2，不应再次播放 `0004.mp3`
6. 当目标重新稳定识别到时，不应播放 `0005.mp3`
7. 此时只应播放当前目标对应的名称语音
8. 如果画面有轻微抖动但一直没真正稳定找回，不应反复连续播放 `0007.mp3`

### 语音测试键测试

1. 按一次 `PE14`，播放 `0001.mp3`
2. 再按一次 `PE14`，暂停播放
3. 再按一次 `PE14`，继续播放

说明：

- `PE14` 只是测试播放控制，不改变当前目标模式
- 如果系统语音先触发了，`PE14` 的测试状态机会被重置

## 8. 代码对应位置

如果你后面要自己改，可以优先看这些位置：

- `Core/Src/main.c`
  - 按键定义区：`PC0 / PC1 / PE13 / PE14 / PE15`
  - DFPlayer 文件编号配置区：`0001 ~ 0007`
  - 系统语音保护参数：`VOICE_MODE_GUARD_MS`、`VOICE_STATUS_GUARD_MS`
  - `Key_Scan_And_Handle()`：模式切换和按键处理
  - `Play_ModeSwitch_Voice()`：模式切换播报入口
  - `Handle_SystemVoice_Update()`：搜索/发现目标自动播报
  - `Handle_Voice_Test_Key()`：`PE14` 测试按键逻辑
