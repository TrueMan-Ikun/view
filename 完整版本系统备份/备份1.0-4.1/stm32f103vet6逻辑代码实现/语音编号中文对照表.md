# 语音编号中文对照表

这份文件用于记录 DFPlayer 当前使用的语音文件编号与中文播报内容，方便后续更换 MP3 文件时核对。

## 当前对应关系

| 文件编号 | 文件名 | 中文播报内容 |
| --- | --- | --- |
| 1 | `0001.mp3` | 厕所在这 |
| 2 | `0002.mp3` | 安全出口在这 |
| 3 | `0003.mp3` | 办公室在这 |
| 4 | `0004.mp3` | 正在搜索目标 |
| 5 | `0005.mp3` | 已发现目标 |
| 6 | `0006.mp3` | 已停止识别 |
| 7 | `0007.mp3` | 目标丢失 |

## 使用提醒

- TF 卡里实际文件名仍然建议保持为 `0001.mp3 ~ 0007.mp3`
- 不建议在 TF 卡里直接使用中文文件名
- 如果只是更换播报内容，不改编号顺序，就不需要改代码
- 当前代码中的编号顺序已经按上表接入
- 如果要让“目标丢失”生效，需要把 `0007.mp3` 也放进 TF 卡的 `MP3` 文件夹

## 当前语音顺序

对于厕所 / 安全出口 / 办公室这三种目标，当前系统播报顺序是：

1. 选择目标后，不立即播报名称
2. 进入搜索时，播放 `0004.mp3`
3. 稳定识别到目标后，播放 `0005.mp3`
4. 然后播放对应名称语音

对应关系如下：

- 厕所模式：`0004 -> 0005 -> 0001`
- 安全出口模式：`0004 -> 0005 -> 0002`
- 办公室模式：`0004 -> 0005 -> 0003`
- 停止识别模式：直接播放 `0006`

## 丢失后重新找回的语音顺序

当系统已经锁定过目标，后来又确认目标丢失时，当前顺序是：

1. 播放 `0007.mp3`
2. 进入重新搜索，但不再播放 `0004.mp3`
3. 重新稳定识别到目标后，不再播放 `0005.mp3`
4. 直接播放当前目标对应的名称语音

对应关系如下：

- 厕所模式：`0007 -> 0001`
- 安全出口模式：`0007 -> 0002`
- 办公室模式：`0007 -> 0003`
